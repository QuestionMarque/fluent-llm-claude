# orchestration/

Control tower of the execution engine. Coordinates the full end-to-end flow
from IR acquisition through planning, execution, and state update.

---

## Modules

### `execution_loop.py`
`ExecutionLoop` — the main orchestrator.

Implements the complete IFU → execution pipeline:

```
obtain IR  (library or LLM)
    ↓
WorkflowDecomposer.decompose(ir)  → Workflow
    ↓
ValidatorWrapper.validate_workflow(workflow)  → ValidationFeedback
    ↓
  [invalid?]
    ├── library mode → fail fast immediately (attempts=1)
    └── llm mode    → build retry prompt, loop back (up to max_retries)
    ↓
  [valid]
for each step:
    RuntimeCall.from_step(step, registry) → RuntimeCall (1:1 lookup)
    PyFluentAdapter.execute(...)   → ExecutionResult
    StateManager.update(step, ok)  → mutate State
    append execution_log entry
    ↓
ExecutionLoopResult {
    success, attempts, workflow, runtime_calls,
    execution_log, state, validation_feedback,
    error, retry_prompt
}
```

### `/ir_examples`
Pre-defined IR examples in JSON format for deterministic testing and demo runs.

**Available workflows:**

| Name | Description |
|------|-------------|
| `pipetting_cycle` | Get tips → aspirate → dispense → drop tips |
| `just_tips` | Pick up tips with airgap configuration |
| `transfer_samples_plate_to_plate` | Aspirate from sample plate, dispense to dilution plate |
| `transfer_samples_tubes_to_plate` | Aspirate from tube runner, dispense to dilution plate |
| `dilute_samples_from_tube` | Multi-step dilution from tubes to plate |
| `serial_dilution` | 10-column serial dilution across a 96-well plate |
| `dilute_samples_from_full_plate` | Diluent distribution + sample transfer for full plate |
| `sample_tube_to_plate_replicates` | Tube to plate with replicates |
| `fill_plate_hotel` | Move plate to hotel, fill, return |
| `distribute_antisera` | Multi-tip antisera distribution |
| `distribute_antigens` | Full-plate antigen distribution |
| `add_tRBC` | tRBC mixing and distribution |

All entries are validated against `STEP_SCHEMA` in the test suite.

## Library Mode vs LLM Mode

### Library Mode (`ir_mode="library"`)
- IR is loaded from `IR_examples/`
- Library IRs are authored — they must always be valid
- If validation fails: **fail immediately** (no retries, `attempts=1`)
- Use for production workflows and deterministic testing

### LLM Mode (`ir_mode="llm"`)
- IR is generated by `llm_client.generate_ir(prompt)`
- LLM output may be imperfect on the first attempt
- If validation fails: build a corrective retry prompt using `FeedbackBuilder`
  and loop back up to `max_retries` times
- Use for natural-language IFU ingestion

## ExecutionLoopResult

```python
@dataclass
class ExecutionLoopResult:
    success: bool
    attempts: int
    workflow: Optional[Workflow]
    runtime_calls: List[RuntimeCall]
    execution_log: List[dict]       # one entry per step
    error: Optional[str]
    retry_prompt: Optional[str]     # set on llm mode exhaustion
    state: Optional[State]
    validation_feedback: Optional[ValidationFeedback]
```

## Minimal Usage

```python
from execution_engine.capability_registry.loader import load_default_registry
from execution_engine.validation.validator_wrapper import ValidatorWrapper
from execution_engine.runtime.pyfluent_adapter import PyFluentAdapter
from execution_engine.orchestration.execution_loop import ExecutionLoop
from main import FluentRuntime


registry = load_default_registry()

loop = ExecutionLoop(
    registry=registry,
    runtime_adapter=PyFluentAdapter(runtime=FluentRuntime()),
    validator=ValidatorWrapper(registry=registry),
    ir_mode="library",
    max_retries=3,
)

result = loop.run(ir_name="pipetting_cycle")
print(result.success)               # True
print(len(result.runtime_calls))    # 4
print(result.state)                 # State(tip_loaded=False, well_volumes={...}, execution_history=[...])
```
